# MVP 开发计划（可执行）

## 目标
在 4 周内交付可演示的离线版桌面虚拟宠物 MVP。

## Week 1: 工程与运行时基础
- 完成工程骨架（App/Shell/Engine/UI/Infra）
- 跑通透明覆盖层窗口
- 跑通随机活动占位宠物（当前已完成基础版）
- 接入基础日志与性能计时

交付物：
- 可运行 Demo（随机活动）
- 工程目录稳定

## Week 2: 物理与平台感知
- 接入窗口扫描（CGWindow API）
- 构建平台图（ground + window top）
- 增加跳跃可达性判定
- 实现失败下落与回落到 ground

交付物：
- 跨平台跳跃最小闭环

## Week 3: 行为与动画联动
- 随机行为引擎接入动作冷却
- 接入动画状态机与过渡
- 用户点击/拖拽事件触发重采样

交付物：
- 随机行为 + 动画可视化联动

## Week 4: 数据、稳定性、验收
- 接入本地资产索引（SQLite）
- 完成“清空本地数据”能力
- 压测 CPU/内存，执行降级策略
- 完成一次 2 小时稳定性跑测

交付物：
- MVP 验收报告
- 发布前问题清单

## 验收门槛（MVP）
- CPU 平均 <= 5%
- 内存 <= 200MB
- 全离线可运行
- 卸载后无业务数据残留
- 随机行为可观察且无固定巡逻路径
